PII tokenization

De-identify in your boundary — and the whole fleet opens up.

Bastion detects PII inside your boundary and swaps it for tokens before a single byte leaves. The model sees [[EMAIL_1]], never the address — and the real values are rehydrated locally in the response. Because what leaves is scrubbed, the offshore-jurisdiction penalty is waived, so confidential work can route to the cheapest capable model anywhere — not just the local or US-frontier tier.

  • PII never leaves — detect → tokenize → send → rehydrate, all in-boundary
  • Unlocks aggressive routing — confidential work becomes eligible for cheap near-frontier models (GLM · Qwen · Kimi)
  • Eyeball or hand-edit the tokenized text before anything is sent
  • Regulated data is never auto-sent — it stays in your boundary, period

Governed web access

Your local model, but it can see the live web.

A local LLM is frozen at its training cutoff — fine for private work, useless for "what shipped this quarter." Bastion gives it governed, in-boundary web access: it runs a self-hosted SearXNG search, fetches and reranks the actual result pages, and answers from fresh sources — without your prompt ever touching a cloud search API. The query itself is classified and egress-governed, so it captures live-data traffic a dumb chatbot would have to hand to a frontier model.

  • Live answers from a local model — no frontier API required to stay current
  • In-boundary deep retrieval — fetch · extract · rerank result pages, all on your infra
  • Egress governor — public/internal queries leave; confidential/regulated are blocked
  • Auto-detects when a prompt actually needs fresh data — no wasted searches

How it works

Classify. Enforce the boundary. Route to the cheapest model that clears the bar. Audit everything.

Every prompt runs the same four-step lifecycle — fail-closed at each gate, so the safe path is the default path.

Classify

Each prompt is scored public · internal · confidential · regulated (rules + optional Microsoft Presidio NER), with explainable, redacted signals.

Enforce the boundary

Regulated & confidential data is hard-blocked to your boundary. Structurally, fail-closed, tested to zero leaks over a corpus.

Route by capability + cost

The cheapest model that clears the quality bar for the task category. Local-first; escalate to cloud only when it actually pays off.

Audit & learn

Every decision lands in a SHA-256 hash-chained trail. Flag a bad call and the correction sticks — and survives a restart.

Why it matters

Your best AI work is your most sensitive — and that's exactly where you can't use the best models.

🔒

Regulated data can't touch a frontier API

HIPAA, GDPR, PCI, and financial data legally can't leave your boundary. So the most capable models stay off-limits for the work that matters most.

💸

One frontier model for everything is expensive

Sending every prompt to a top-tier US model burns budget on work a local or near-frontier model would handle just as well — at a fraction of the price.

🕳️

Black-box routing has no proof

If you can't show which model saw which data — and prove regulated data never left — you can't pass an audit, and you can't trust the savings.
